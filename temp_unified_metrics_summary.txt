  - Many unified* scripts produce objective vectors where:
    index0 names sample: ['qtd_doors']
    index1 names sample: ['iterations']
    index2 names sample: ['distance']
  - From return statements and obj usage, the typical objective vector is:
      [num_doors, iterations_or_avg_iters, distance]
    where "iterations_or_avg_iters" is an iteration count (not wall-clock time).
  - There is little evidence of an explicit wall-clock "tempo_total" objective in unified*; iterations were used instead as the time-like metric.
